Historia dos computadores

Historia Real
voltar

A historia do computadores começou la na epoca da guerra, mas o computador que foi projetado para ja começar a fazer grandes calculos, é o Harvard Mark (1944), ele era eletromecanico baseado em calculos assim iniciando a 1ª Geração de computadores. La por meados dos anos 40 o Colossus (1946), que era um computador mais moderno para epóca que fazia cripto analise, ele era usado pela Alemanha para calculos mais complexos. Só que ao mesmo tempo tinha nos EUA o ENIAC (1946), que era automatico e fazia os calculosde forma rapida e moderna, surgiu nesta epoca o termo bug que a tradução literal é incetos, pois eles sentavam nas partes eletronicas do computador e causava alguns problemas. A 2ª Geração (1959 - 1965) tivemos uma revolução com computadores ainda menores que os anteriores, agora usando transistores. A 3ª Geração (1965 - 1970) teve a modernização dos aparelhos circuitos integradoos, trnaistores, capacitores, resistores. E entramos na 4ª Geração (1971- Atualmente) que é já era atual dos computadores, com aparelhos cada vez menores e é marcada pelos PC os computadores portateis. Assim surgindo a internet durante a guerra fria para melhorar os meios de comunicações.

Clients

voltar

Temos que intender o conceito e temos dois lados dessa historia, o lado Clients e o lado do servidor Server.

O que temos do lado do Client são as pessoas que consomem informações da internet, e esse outro lado é o Server que é por onde faz a comunicação entre redes dormando uma rede de cominicação. Os navegadores são ferramentas utilizadas para os clients ter acesso a internet. São programas criados por empresas, utilizados para abrir e executar arquivos e seguem um padrão W3C, porem sempre tem uma diferença ou outra de interpretação. É Também chamado de "Browser", começõu com o MOSAIC, passou para o Netsiqe e hoje temos uma variedade de navegadores disponiveis gratuitamente. Aplicações Web são resoluções criadas que possuem a internet como meio de comunicação entre Client X Server não sendo necessaria sua instalação, um exemplo que pode ser e não ser Aplicações Web é o facebook, se ele for acessado pelo aplicativo ele nâo é uma Aplicação web pois ele tem que ser feito dowload, agora ele acessado pelo navegador onde não é necessario um download de arquivo, ele se tonra uma Aplicação Web.

Servers

voltar

O Server(servidor) sãocomputadores centrais em um determinado lugar, onde vai fazer a comunicação com o os clients. Estes servidores não preisam necessariamente estar conectados com a internet, ele pode ser um compudator em uma empresa por exemplo, para guardar os documentos e dados internos da empresa isso é chamado de rede interna. Os Servidores tem de ser mais robustos pois ele precisa fazer o processamento de varios outros dispositivos conectados nele, ele tem de servir a varios outros computadores fazendo requisições para ele.

Existem varios tipos de servidores:

  • De Arquivos
  • De Segurança (Firewall)
  • Streaming
  • E-mail
  • Web
O Servidor é dividido em duas parte, a parte loógica e a parte física:
  1. Software
    • Sistemas Operacionais
    • Monitoramento
    • Servidores Web
  2. Hardware
    • Armazenamento (isco rígido, SSDs)
    • Memória
    • Processadores

Estutura basica de HTML

voltar

Todo site em HTML precisa de uma estrutura padrão para funcionar. Pense nisso como o "esqueleto" da página. Sem essas tags, o navegador não sabe como ler o seu código.

Aqui está o código mínimo obrigatório:

HTML


<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <title>Título da Página</title>
</head>
<body>

  <!-- Todo o conteúdo visível do site entra aqui -->

</body>
</html>

Explicando cada parte em poucas palavras:

  • !DOCTYPE html: Avisa ao navegador que o site usa a versão mais moderna (HTML5).
  • html lang="pt-BR": Abre o documento e avisa que o site está em português.
  • head: A "cabeça" do site. Aqui ficam as configurações escondidas, como o título da aba (title) e os links para o CSS.
  • body: O "corpo" do site. É aqui dentro que você coloca tudo o que o usuário realmente vai ver na tela (textos, imagens, links).

Listas Ordenadas e Listas Não Ordenadas

voltar

No HTML, usamos listas para organizar itens. A diferença entre elas é puramente a presença ou não de uma ordem numérica.

  1. Lista Não Ordenada (ul)
      2. É a lista com marcadores (as famosas "bolinhas"). Usamos quando a ordem dos itens tanto faz.
    • Tag principal: ul (Unordered List)
    • Tag de cada item: li (List Item)
  2. Lista Ordenada (ol)
      3. É a lista numerada (1, 2, 3...). Usamos quando a ordem dos fatores importa, como em um passo a passo ou ranking.
    • Tag principal: ol (Ordered List)
    • Tag de cada item: li (List Item)
Regra de ouro: Não importa o tipo da lista (ul ou ol), o conteúdo que vai lá dentro sempre tem que estar envelopado pela tag.
